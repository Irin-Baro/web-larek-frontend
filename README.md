## 🏗️ Архитектура <a id=mvp></a>

### **1. MVP (Model-View-Presenter)**
Проект построен по архитектурному паттерну MVP:

- **Model** - Типы данных (`Product`, `CartItem`, `Order`) в `src/types/`
- **View** - UI-компоненты (`ProductCard`, `Cart`, `OrderForm`) наследуются от `Component`
- **Presenter** - Соединительное звено (`src/app/app.ts`), которое подписывается на события View, вызывает методы в Model и передает обновленные данные обратно во View.

**Принципы MVP:**
- View отвечает только за отображение
- Presenter содержит всю бизнес-логику
- Model содержит только данные
- View и Presenter не зависят друг от друга напрямую

### **2. Observer (Наблюдатель) / EventEmitter**
Для связи между компонентами используется паттерн Наблюдатель через `EventEmitter`:

- Компоненты генерируют события
- `EventEmitter` выступает в роли брокера событий
- Presenter подписывается на интересующие события
- Слабая связность между компонентами

### **3. Component (Компонент)**
Используется паттерн `Шаблонный метод`, где определяет общую структуру (конструктор, методы работы с DOM), а наследники реализуют специфичную логику в абстрактном методе init() и переопределяют render().
Абстрактный класс `Component`:
- Инкапсулирует логику работы с DOM
- Предоставляет единый интерфейс для всех UI-компонентов
- Позволяет переиспользовать код через наследование

## 🔧 Установка и запуск<a id=run></a>

1. **Клонируйте репозиторий:**
   ```bash
   git clone git@github.com:Irin-Baro/web-larek-frontend.git
   cd web-larek-frontend
   ```
2. **Установите зависимости:**

    ```bash
    npm install
    ```
3. **Настройте окружение:**

    Создайте файл .env в корне проекта:

    ```bash
    API_ORIGIN=https://larek-api.nomoreparties.co
    ```
4. **Запустите в режиме разработки:**

    ```bash
    npm start
    ```
    или

    ```bash
    yarn
    yarn start
    ```

    Приложение будет доступно по адресу:
    ```bash
        http://localhost:8080
    ```
5. **Сборка для production:**

    ```bash
    npm run build
    ```
    или

    ```bash
    yarn build
    ```

## 📦 Ключевые классы<a id=class></a>
- **EventEmitter** - Центральный брокер событий для связи компонентов без прямых зависимостей.

    **Расположение:** `src/components/base/events.ts`

    **Основные методы:**
    - `on(event, callback)` - подписка на событие
    - `emit(event, data)` - публикация события
    - `off(event, callback)` - отписка от события

    **Типы событий**
    Все события приложения описаны в `src/types/events.ts`:

    ```typescript
    // Пример типизации событий
    export type AppEvent = 
        | 'catalog:changed'
        | 'product:select'
        | 'cart:add'
        | 'cart:changed'
        | 'order:submit';
    ```

- **Component** - Абстрактный базовый класс для всех UI-компонентов.

    **Основные методы:**

    ***render()*** - возвращает DOM-элемент

    ***setText()*** - установка текста

    ***setImage()*** - установка изображения

    ***setEventListener()*** - добавление обработчика событий

- **ProductCard** - Компонент карточки товара для отображения в каталоге.

    **Расположение:** `src/components/ui/ProductCard/ProductCard.ts`

    **Назначение:**
    - Отображение информации о товаре в каталоге
    - Обработка кликов пользователя
    - Генерация событий для Presenter

    **Методы:**
    - `render()` - заполняет DOM данными товара
    - `init()` - навешивает обработчики событий

    **Принципы работы:**
    - **Пассивность** - только показывает данные и генерирует события
    - **Переиспользование** - наследует общую логику от `Component`
    - **Изоляция** - не знает о других компонентах и бизнес-логике

- **App (Presenter)** - Главный класс приложения, реализующий бизнес-логику.

    **Расположение:** `src/app/App.ts`

    **Назначение:**
    1. **Приём пользовательского ввода** - через EventEmitter
    2. **Бизнес-логика** - обработка событий, работа с данными
    3. **Координация компонентов** - создание и управление View
    4. **Работа с сервисами** - API, localStorage, etc.

| **Метод**   | **Endpoin**   |**Описание**|

|POST    |`/order`   |Оформление заказа|


## 🎯 События приложения<a id=app></a>

#### 📁 Каталог
> `catalog:changed` — обновление списка товаров

> `product:select` — выбор товара для просмотра

#### 🛒 Корзина
> `cart:add` — добавление товара в корзину

> `cart:remove` — удаление товара из корзины

`cart:changed` - изменение состояния корзины

#### ✔️ Заказ:
> `order:submit` - отправка заказа

> `order:success` - успешное оформление

## 🔄 Поток данных<a id=data></a>

### Загрузка приложения:
- **App** инициализирует `EventEmitter` и `API`
- Загружает каталог товаров с сервера
- Восстанавливает корзину из `localStorage`

### Взаимодействие пользователя:
1. Клик на "Купить" в `ProductCard`
2. `ProductCard` генерирует событие `cart:add`
3. **App** обрабатывает событие, обновляет корзину
4. **App** генерирует событие `cart:changed`
5. Все компоненты, зависящие от корзины, обновляются
